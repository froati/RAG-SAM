[삼성전자 AI 투자 비서] RAG 엔진 최적화 비교 보고서

1. 개요
본 보고서는 초기 RAG(Retrieval-Augmented Generation) 시스템과 현재 최적화된 '절충안' 시스템의 성능 및 로직 차이를 분석하여 기록합니다.

2. 성능 지표 비교 (RAGAS 평가 기준)

| 평가지표 | 초기 버전 (Initial) | 최적화 버전 (Compromise) | 변화 정도 |
| :--- | :--- | :--- | :--- |
| Faithfulness (성실성/정확도) | 약 0.35 | 0.63 | +80% 향상 |
| Answer Relevancy (답변 관련성) | 약 0.65 | 0.55 | -15% 하락 |
| Context Precision (검색 정밀도) | 약 0.55 | 0.70 | +27% 향상 |
| Context Recall (정답 재현율) | 약 0.45 | 0.60 | +33% 향상 |

3. 주요 변경 사항 및 기술적 근거

A. 문서 파싱 및 색인 (Chunking Strategy)
- 변경 내용: 청크 크기 1000자 유지하되, 검색 컨텍스트 양(k)을 5개에서 7개로 확대.
- 이유: 600자로 줄였을 때 재무제표 표 데이터가 잘려 정보가 소실되는 현상을 발견하여 다시 1000자로 원복함. 대신 k값을 늘려 더 넓은 범위의 정보를 AI에게 전달함으로써 정답이 포함될 확률(Context Recall)을 높임.

B. 검색 알고리즘 (Search Strategy)
- 변경 내용: MMR(Max Marginal Relevance) 시도 후 다시 단순 유사도 검색(Similarity Search)으로 회귀.
- 이유: MMR은 문서의 다양성을 높여주지만, 정답이 특정 페이지에 집중된 공시 보고서의 특성상 오히려 핵심 정보를 뒤로 밀어내는 역효과를 초래함. 단순 유사도 검색이 수치 데이터를 찾는 데 더 효율적임을 확인.

C. 프롬프트 엔지니어링 (Prompting)
- 변경 내용: '직접적인 문장이 없더라도 본문의 수치들을 조합하여 논리적으로 설명하라'는 지침 추가.
- 이유: 너무 엄격한 제약(문서에 없으면 답하지 마라)은 AI를 소극적으로 만들어 답변 거부를 유도함. AI의 분석 능력을 활용해 매출 합산이나 전년 대비 증감률을 스스로 계산할 수 있도록 유연성을 부여함.

4. 종합 결론 및 분석

초기 버전은 답변이 풍부하고 유려했으나, 실제 문서에 근거하지 않은 '할루시네이션(환각)' 수치가 다수 포함되어 금융 서비스로서의 신뢰도가 낮았습니다.

최적화된 현재 버전은 다음과 같은 이점이 있습니다:
- 높은 신뢰도: Faithfulness가 비약적으로 상승하여 사용자가 믿고 사용할 수 있는 데이터를 제공합니다.
- 정교한 검색: Context Precision/Recall의 상승으로 질문에 필요한 핵심 데이터를 놓치지 않고 잘 찾아옵니다.
- 분석적 답변: 단순 요약을 넘어 문서 내 수치들을 연결하여 설명하는 능력이 강화되었습니다.

비록 Answer Relevancy가 소폭 하락했으나, 이는 AI가 지어내는 말(Hallucination)이 줄어들면서 발생하는 현상으로, 시스템의 전체적인 품질과 안전성은 현재 버전이 월등히 우수하다고 판단됩니다.
