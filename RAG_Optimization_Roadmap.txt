# RAG 시스템 고도화 로드맵 (Investment RAG Optimization Roadmap)

본 문서는 삼성전자 투자 비서 시스템의 성능(RAGAS 지표)을 극대화하기 위한 단계별 고도화 방향을 제시합니다.

## 1단계: 문서 파싱 및 구조화 (Parsing & Structuring)
*   **표(Table) 보존 파싱:** 현재의 단순 텍스트 추출에서 벗어나, 표 데이터를 Markdown 또는 HTML 형식으로 변환하여 추출합니다.
    *   도구: `LlamaParse`, `UnstructuredPDFLoader`, `Marker-PDF`
    *   기대효과: 재무제표의 수치 데이터가 깨지지 않고 LLM에게 전달되어 Faithfulness 점수 대폭 향상.
*   **레이아웃 인식 분할 (Layout-aware Chunking):** 페이지 번호, 제목, 본문, 주석 등을 구분하여 의미 단위로 청크를 생성합니다.
    *   기대효과: 문맥이 끊기는 현상을 방지하여 Context Recall 개선.

## 2단계: 검색 성능 고도화 (Retrieval Enhancement)
*   **하이브리드 검색 (Hybrid Search):** 벡터 검색(Semantic)과 키워드 검색(BM25)을 결합합니다.
    *   이유: 'HBM4', 'DDR5'와 같은 고유 명사 및 특정 수치 검색 시 키워드 매칭이 더 정확할 수 있습니다.
*   **리랭킹 (Re-ranking):** 검색된 상위 k개 문서를 Cross-Encoder 모델(예: BGE-Reranker)을 통해 다시 순위를 매깁니다.
    *   기대효과: 질문과 가장 밀접한 문서를 최상단에 배치하여 Context Precision 점수 향상.

## 3단계: 멀티모달 RAG (Multimodal RAG)
*   **비전 모델 활용 (Vision-based Analysis):** 보고서 내의 차트, 그래프 이미지를 GPT-4o 등 Vision 모델로 직접 분석합니다.
    *   방법: 이미지를 캡셔닝(Captioning)하여 텍스트로 저장하거나, 검색된 페이지 전체 이미지를 모델에게 전달.
    *   기대효과: 텍스트에 없는 시각적 추세(Trend) 정보를 답변에 반영.

## 4단계: 동적 가이드 및 자가 수정 (Self-Correction)
*   **Self-RAG 도입:** 모델이 자신의 답변이 문서에 근거하는지 스스로 판단하고, 부족할 경우 검색을 재시도하는 로직을 추가합니다.
*   **쿼리 변형 (Query Expansion/Rewriting):** 사용자의 모호한 질문을 검색에 최적화된 여러 개의 질문으로 변환하여 검색 범위를 넓힙니다.

## 5단계: 평가 및 모니터링 자동화
*   **CI/CD 기반 평가:** 코드 변경 시마다 Ragas를 통해 자동으로 점수를 측정하고, 기준치 미달 시 배포를 차단하는 파이프라인 구축.
