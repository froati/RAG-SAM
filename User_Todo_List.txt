# 사용자 후속 작업 리스트 (User To-Do List)

RAG 엔진 고도화 업데이트 이후, 성능 향상을 확인하고 다음 단계로 나아가기 위해 사용자가 직접 수행해야 할 작업들입니다.

## 1. 인덱스 초기화 및 재학습 (필수)
현재 `faiss_index` 폴더에는 이전 방식(PyPDF)으로 추출된 데이터가 저장되어 있습니다. 새로운 고도화 파서(PyMuPDF)를 적용하려면 기존 인덱스를 지워야 합니다.
- **작업:** `faiss_index` 폴더를 삭제합니다.
- **방법:** `app.py`를 다시 실행하면 자동으로 새로운 방식으로 PDF를 분석하고 인덱스를 생성합니다.

## 2. 수치 답변 정확도 재검증
이전에 답변에 실패했던 질문들을 다시 던져보고 답변의 질이 어떻게 변했는지 확인합니다.
- **테스트 질문 예시:**
    - "2026년 1분기 연결 기준 매출액과 영업이익은 각각 얼마인가요?"
    - "보통주 1주당 지급되는 분기 배당금은 얼마입니까?"
- **확인 포인트:** 답변 내에서 "보고서 p.XX에 따르면"과 같이 정확한 출처(페이지 번호)가 명시되는지 확인하세요.

## 3. RAGAS 성능 평가 재실행
엔진 업데이트가 실제로 지표(Metrics)를 얼마나 올렸는지 수치로 확인합니다.
- **작업:** 터미널에서 `python "ragas 평가/ragas_eval.py"`를 실행합니다.
- **비교:** 새로 생성된 `evaluation_results.csv`의 점수와 이전 점수(Faithfulness 등)를 비교해 보세요.

## 4. 고급 파싱 도입 여부 결정 (다음 단계)
현재 업데이트한 PyMuPDF보다 더 강력한 표 분석이 필요하다면 다음 도구 도입을 검토해야 합니다.
- **LlamaParse 도입:** 표를 완벽한 Markdown으로 변환해 줍니다. (LlamaIndex 계정 및 API 키 필요)
- **도입 결정 시:** `rag_engine.py`에서 로더 부분을 `LlamaParse`로 교체하는 작업을 요청해 주세요.

## 5. 추가 보고서 데이터 확보
현재 `data` 폴더에 있는 파일들 외에 추가적인 삼성전자 공시 보고서(예: ESG 보고서, 연간 보고서)가 있다면 폴더에 넣어 학습 범위를 넓혀보세요.
